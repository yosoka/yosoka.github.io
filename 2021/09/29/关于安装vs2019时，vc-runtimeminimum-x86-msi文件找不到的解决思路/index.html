关于安装vs2019时，vc_runtimeminimum_x86.msi文件找不到的解决思路

发表于 2021-09-29 更新于 2021-10-23 分类于 日常分享 Waline： 阅读次数： 本文字数： 1.3k 阅读时长 ≈ 1 分钟

以下只是个人观点，可能有不对的地方，请指正！ありがとうございます

一.初遇问题

刚开始，是在下载Adobe Photoshop的时候，看到了如下的弹窗

后来安装Visual Studio 2019的时候也遇到了如上弹窗

开始意识到问题的存在

二.开始寻找问题所在

我点开了弹窗下方的文件路径查询，发现它需要一个名字是vc_runtimeminimum_x86.msi的文件

我就搜索，查到了这些教程 这是B站的 这是简书的 大体讲述的方法都是一样的

第一步

完全删除有关**Visual Studio c++**的所有程序

这里的完全删除，我一直无法做到，个别程序残留无法清除，甚至根本找不到卸载的exe文件

作者还有提供一个微软官方的卸载工具 这是链接 这个直接就是下载链接，打开即可下载

下载完成之后无需安装，直接打开即可使用

当然你也可以选择修复，修复出现问题的版本

这个方法我还没有具体尝试过，因为我根本无法安装

第二步

卸载完成后（一定是卸载完成之后），如果卸载不干净的话，之后都是无用功

确认完成之后，开始下载所有版本的Visual Studio c++

没错，就是卸载了重装，就是这么简单

可能我比较特殊吧，我的事情远远没有这么简单

这里是微软官方的 Visual C++ 下载地址

用这个下载会比较麻烦，因为要一个版本一个版本的下载

这里推荐用第三方的集成包

360提供的64位版：更新至2020.3

360提供的32位版：更新至2020.3

下载集成包可以一次性全部安装，比较方便

以上，如果你没有意外，应该已经完成了问题的解决

三.无法解决的问题

对的，我的问题还没有得以解决

我查找了大量的教程，提交了报错的代码，但是回复实在慢，于是我又去了淘宝，找到了修电脑的师傅，远程查看了问题

PS : 问题可以和客服交流，没有解决是可以退钱的

师傅，尝试了一番，说是系统的问题，系统错误

于是我又查了教程，发现可能和注册表残余有关（不知道对不对）

再次联系师傅，师傅说需要重装系统，或者花50元，店家提供排查服务

四.解决问题

因为有关系统，主要确实有点贵，我就自行研究了一番

之前师傅有提及要重装系统，不过被我拒绝了

但最后我还是妥协了，因为实在找不到比较好，且具性价比的方法了

我找了三种比较实用的系统恢复方法

  • 连续按F2进入BIOS界面【Security】菜单，通过方向键选择【Secure Boot 】选项，点击【Secure Boot Control】选项，将其设定为 “Disabled”，通过方向键进入【Boot】菜单CSM support 改为enable 找到launch PXE OpROM policy 改成enable，通过方向键进入【Advanced】菜单通过方向键选择【SATA Congiguration】选项，将硬盘格式改为【AHCI】,按F10保存退出。

  • 建立及使用U盘的安装介质重新安装系统 这里以华硕为例

  • 打开系统设置，在Windows更新页面，打开高级设置，往下找到恢复，并按照提示操作即可

    PS：这个方法只能恢复当前系统，比如我是win11，那么恢复完仍然是win11，如果想要安装其他系统不推荐用这个

恢复完系统，问题终于得到了解决

以上就是我对vc_runtimeminimum_x86.msi文件找不到的所有解决思路
